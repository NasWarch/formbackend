Documentation

Configurez Monito en 5 minutes.

Démarrage rapide

  1. Créez un compte gratuit — pas de carte bancaire
  2. Depuis le dashboard, cliquez sur "+ Nouveau monitor"
  3. Entrez l'URL de votre serveur et choisissez le type de check
  4. Configurez vos notifications (email, Slack, Telegram)
  5. Et c'est tout ! Les alertes arrivent automatiquement en cas de panne.

Types de checks

HTTP/HTTPS
Check HTTP

Vérifie le code HTTP, le temps de réponse, les redirections. Configurez un timeout et des mots-clés attendus.

PING
Ping ICMP

Mesure la latence et la perte de paquets. Idéal pour vérifier qu'un serveur est joignable sur le réseau.

TCP
Port TCP

Vérifie qu'un port spécifique est ouvert et accepte les connexions. Parfait pour SSH (22), MySQL (3306), etc.

Notifications

Email Notifications par email SMTP. Configurable depuis le dashboard.
Slack Webhook Slack — recevez les alertes dans votre canal #incidents.
Telegram Bot Telegram privé ou groupe. Configuration via token bot.
Webhook Endpoint HTTP personnalisé. Format JSON documenté ci-dessous.

Référence API

GET /api/monitors

Liste tous vos moniteurs avec leur statut actuel. Authentification requise.

GET /api/monitors/{id}

Détail d'un moniteur : historique uptime, incidents, temps de réponse.

POST /api/monitors

Crée un nouveau moniteur. Body : name, type (http|ping|port), target, interval, timeout.

DELETE /api/monitors/{id}

Supprime un moniteur et son historique.

Payload Webhook

Lorsqu'un incident est détecté ou résolu, nous envoyons un POST à votre webhook :

{
  "event": "incident.opened" | "incident.resolved",
  "monitor": {
    "id": "m_a1b2c3d4",
    "name": "Serveur production",
    "type": "http",
    "target": "https://example.com"
  },
  "incident": {
    "id": "inc_xyz",
    "started_at": "2026-05-07T10:30:00Z",
    "resolved_at": null,
    "duration_seconds": null,
    "status_code": 503,
    "error": "HTTP 503 Service Unavailable"
  }
}

Déploiement Docker (Self-Hosted)

# docker-compose.yml
version: '3.8'
services:
  app:
    image: monito/app:latest
    ports:
      - "8000:8000"
    environment:
      - DATABASE_URL=postgresql://...
      - REDIS_URL=redis://redis:6379
    depends_on:
      - redis
  redis:
    image: redis:7-alpine
