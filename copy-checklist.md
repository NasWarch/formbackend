# Copy Checklist — Template Réutilisable

> Tenant : monetization-lab
> Usage : checklist à appliquer à chaque page, email, ou message d'acquisition.
> Valable pour landing page, emails transactionnels, campagnes, docs, et messages in-app.

---

## 1. Principes Fondamentaux

- [ ] **Un message = une idée.** Si une phrase en contient deux, divise-la.
- [ ] **Lis-toi à voix haute.** Si ça sonne faux à l'oral, réécris.
- [ ] **Pas d'adjectifs vides.** « Innovant », « révolutionnaire », « puissant » → supprimer ou remplacer par un fait.
- [ ] **Le client est le héros, pas le produit.** Le produit est l'outil.
- [ ] **Chiffre > adjectif.** « 2x plus rapide » plutôt que « très rapide ».
- [ ] **Verbes d'action.** « Obtenez votre rapport en 30 secondes » > « Notre outil génère des rapports ».

---

## 2. Par Type de Contenu

### Landing Page — Section par Section

| Section | Règle | Check |
|---------|-------|-------|
| Titre H1 | ≤ 10 mots, bénéfice clair, pas de slogan creux | ☐ |
| Sous-titre H2 | Précise le bénéfice : « En combien de temps / pour qui / quel résultat » | ☐ |
| CTA | « Verbe + résultat concret » — pas « En savoir plus » sans contexte | ☐ |
| Description features | Chaque ligne : « Vous pouvez X pour obtenir Y » | ☐ |
| Témoignages | « [Rôle] chez [Entreprise] : citation avec résultat chiffré » | ☐ |
| FAQ | Question = objection client réelle (pas une question que tu veux qu'on pose) | ☐ |

### Emails Transactionnels

- [ ] Objet : ≤ 50 caractères, pas de mot spammy (gratuit, promo, urgent)
- [ ] Preview text (pré-header) : résume l'email en ≤ 100 caractères
- [ ] Personnalisation : prénom dans les 2 premières lignes
- [ ] Un seul CTA par email (si plusieurs, priorité claire visuellement)
- [ ] Lien de désabonnement visible (obligatoire légal)
- [ ] Signature : prénom + prénom + lien produit (pas une boîte no-reply@)

### Emails d'Acquisition (cold outreach)

- [ ] Objet : personnalisé, pas de template évident
- [ ] Première ligne : référence à quelque chose de spécifique au destinataire
- [ ] Valeur donnée avant valeur demandée : un conseil, un accès, un compliment sincère
- [ ] Longueur : 5-8 lignes max. Pas de pavé.
- [ ] CTA : une question simple, pas un lien (« Ça te dit qu'on en parle 10 min ? »)
- [ ] Post-scriptum optionnel : lien vers une ressource utile (pas le produit)

### Messages In-App

- [ ] Ton : même voix que le produit, pas de rupture de style
- [ ] Contexte : le message apparaît au bon moment (après une action, pas à l'ouverture)
- [ ] Actionable : un bouton ou un lien, pas juste une info
- [ ] Fermable : possibilité de dismiss sans perte

## 3. Anti-Patterns à Éviter

| Anti-pattern | Exemple | Correction |
|---|---|---|
| **Jargon** | « Solution holistique de synergie data-driven » | « Un tableau de bord qui suit vos métriques clés » |
| **Généralité** | « Nous aidons les entreprises à mieux travailler » | « Nous réduisons le temps de reporting de 4h à 15min » |
| **Hyperbole** | « Le meilleur outil du marché » | Preuve ou suppression |
| **Passif** | « Vos données sont protégées » | « Nous chiffrons vos données de bout en bout » |
| **Fausse urgence** | « Offre limitée, dépêchez-vous » (sans vraie limite) | « Les 100 premiers inscrits ont -30% à vie » |
| **Seconde personne sans empathie** | « Vous devez optimiser vos processus » | « Vous perdez du temps sur X ? Voici comment Y vous aide » |

---

## 4. Test de Clarté (B2B spécifique)

- [ ] Un newbie du secteur comprend-il la valeur en 5 secondes ?
- [ ] Le produit est-il décrit sans comparatif ? (« C'est comme X mais pour Y » → acceptable si X est connu)
- [ ] Peut-on supprimer 30% des mots sans perdre le sens ?
- [ ] Y a-t-il un mot que tu n'utiliserais pas avec ta mère / ton meilleur ami ? Supprime-le.
- [ ] Chaque phrase passe-t-elle le test du « Et alors ? » → (pourquoi le client devrait s'en soucier)

---

## 5. Checklist SEO (On-Page)

- [ ] H1 unique et pertinent
- [ ] H2, H3 hiérarchisés
- [ ] Meta description : 150-160 caractères, inclut mot-clé + CTA
- [ ] Alt text sur toutes les images
- [ ] Internal links vers pages clés (pricing, docs, blog)
- [ ] URL courte (≤ 60 caractères, mots-clés uniquement)
- [ ] Structured data (JSON-LD) : Product, FAQ, Review, Organization

---

> **Règle d'or :** relire 24h après écriture. Si une phrase ne te plaît plus, c'est qu'elle n'était pas assez bonne.
